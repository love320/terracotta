PATCH HISTORY

-------------
Patch Level 1
- - https://jira.terracotta.org/jira/browse/DEV-1875
- Fix for Cluster Statistics Recorder where it was not respecting the bind configuration
  parameter and could not connect to the server.

-------------
Patch Level 2

- https://jira.terracotta.org/jira/browse/CDV-874
- The JMX connection URL created by the AdminConsole replaces the passed in server hostname 
  with the actual host addresss. This bypasses ssh tunneling. It's a simple fix to just use the user-specified server host when creating the JMX connection URL

-------------
Patch Level 3

- https://jira.terracotta.org/jira/browse/CDV-882
- made lock refresh cancelable, synchronize the selection
  of server locks with client locks and vice versa, added Inspect lock object
  context menu item.

-------------
Patch Level 4

- https://jira.terracotta.org/jira/browse/CDV-929
-  Interrupted Exception in JMX in the server could cause the server to stall.

- https://jira.terracotta.org/jira/browse/CDV-930
- Fixed "java.lang.OutOfMemoryError: unable to create new native thread" in the server
  caused by The tasks are cancelled, the timer isn't. A new timer is created for each 
  capture session. This could corresponds to the monitoring tool that might be creating 
  capture sessions all the time. 

- http://jira.terracotta.org/jira/browse/CDV-867
-  typo in retrieving properties causing tc-stats.sh throws unknownhostexception

- http://jira.terracotta.org/jira/browse/CDV-868
- typo in retrieving properties from the Commons CLI stuff inside CVT. tc-stats.sh throws numberformatexception

--------------
Patch Level 5

- https://jira.terracotta.org/jira/browse/DEV-1999
- Fix Lock GC recalls lock, nodes hold locks, nodes request locks, then nodes disconnect and then keeps
  waiting for the write lock grant
