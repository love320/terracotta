Terracotta for Spring

Inventory Sample Application

About

The Inventory sample application shows how to use Terracotta for Spring to enable simple, transparent and natural clustering of Spring beans, to enable persistence, fail-over and coherence across a cluster.

Details about what this sample application does and how it works can be found in:  Terracotta for Spring Guide

Build Instructions

This sample application ships built ready to run.  If you modify the application code, use the provided build.xml file to re-build the application with Apache Ant (ant build).

Run Instructions

  1. Start a Terracotta Server from the spring/samples directory.

    Unix: ./start-demo-server.sh
    Windows: start-demo-server.bat
  2. Start one or more inventory clients from the spring/samples/inventory directory

    Unix: ./inventory.sh
    Windows: inventory.bat
