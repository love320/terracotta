Terracotta for Spring

WorkManager Sample Application

About

The WorkManager sample application shows how to use Terracotta for Spring to cluster Spring beans and how to naturally coordinate concurrent execution in the cluster.

Details about what this sample application does and how it works can be found in:  Terracotta for Spring Guide

Build Instructions

This sample application requires Java 5 and needs to be built before you can run it.  Use the provided build.xml file to build the application with Apache Ant (ant build).

Run Instructions

  1. Start a Terracotta Server from the spring/samples directory.

    Unix: ./start-demo-server.sh
    Windows: start-demo-server.bat
  2. Start a WorkManager from the spring/samples/workmanager directory.

    Unix: ./run-work-manager.sh
    Windows: run-work-manager.bat
  3. Start one or more Workers

    Unix: ./run-worker.sh
    Windows: run-worker.bat
